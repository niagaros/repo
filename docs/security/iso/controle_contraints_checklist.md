# Annex A Control Compliance Checklist (ISO 27001)

Use this checklist to verify if the Annex A controls are implemented and effective.

---

## A.5 – Organizational Controls

| Control Reference | Control Description | Check | How to Verify |
|------------------|---------------------|--------|----------------|
| A.5.1 Policies for information security | Policies exist, are approved, reviewed, and communicated. | [ ] | Check policy repository, approval logs, and communication evidence. |
| A.5.2 Information security roles & responsibilities | Roles are defined, assigned, and documented. | [ ] | Check RACI matrix, job descriptions, organizational chart. |
| A.5.3 Segregation of duties | Conflicting roles are separated or controlled. | [ ] | Review access rights and role assignments. |
| A.5.4 Management responsibilities | Management actively supports information security. | [ ] | Review meeting minutes, strategic plans. |
| A.5.5 Contact with authorities | Procedures exist for contacting authorities. | [ ] | Check incident response documentation. |
| A.5.6 Contact with special interest groups | Membership or participation with external security groups. | [ ] | Evidence of participation or membership records. |
| A.5.7 Threat intelligence | Process to collect and analyze relevant threat intelligence. | [ ] | Check threat reports or SOC portal. |
| A.5.8 Information security in project management | Security requirements integrated into projects. | [ ] | Review project documentation, templates, decision gates. |

---

## A.6 – People Controls

| Control Reference | Control Description | Check | How to Verify |
|------------------|---------------------|--------|----------------|
| A.6.1 Screening | Background checks performed before employment. | [ ] | Check HR onboarding procedure. |
| A.6.2 Terms & conditions of employment | Employees sign confidentiality or security clauses. | [ ] | Verify signed agreements and employee files. |
| A.6.3 Security awareness, training, and education | Training is provided and tracked. | [ ] | Check training logs, LMS exports. |
| A.6.4 Disciplinary process | Policy exists for violations. | [ ] | Review HR policy documents. |

---

## A.7 – Physical Controls

| Control Reference | Control Description | Check | How to Verify |
|------------------|---------------------|--------|----------------|
| A.7.1 Physical security perimeters | Areas secured against unauthorized access. | [ ] | Verify access logs, badges, CCTV policies. |
| A.7.2 Physical entry controls | Access restricted and monitored. | [ ] | Check access control system, visitor logs. |
| A.7.3 Securing assets | Assets protected from loss, damage, theft. | [ ] | Review asset inventories and labeling. |
| A.7.4 Protecting against environmental threats | Controls to mitigate flooding, fire, heat, humidity, etc. | [ ] | Check building infrastructure protections. |
| A.7.5 Working in secure areas | Additional controls for sensitive areas. | [ ] | Validate procedures, extra access restrictions. |
| A.7.6 Clear desk and clear screen | Procedures to keep desks/screens secure. | [ ] | Observe workstations, review policies. |

---

## A.8 – Technological Controls

| Control Reference | Control Description | Check | How to Verify |
|------------------|---------------------|--------|----------------|
| A.8.1 User access management | Access granted on a need-to-know basis. | [ ] | Review access logs and user provisioning records. |
| A.8.2 Identity and authentication | MFA, password management, account controls. | [ ] | Validate settings in AD/SaaS tools. |
| A.8.3 Remote access | Remote access secured and monitored. | [ ] | Check VPN logs, remote access policies. |
| A.8.4 System and application hardening | Systems are configured securely and patched. | [ ] | Verify patch reports, config baselines. |
| A.8.5 Backup | Backups exist and are tested. | [ ] | Ask for backup logs, restore test evidence. |
| A.8.6 Logging and monitoring | Security logs exist and are reviewed. | [ ] | Check SIEM dashboard, log retention. |
| A.8.7 Vulnerability management | Vulnerabilities scanned and mitigated. | [ ] | Review scan results, remediation tickets. |
| A.8.8 Change management | Changes are approved and documented. | [ ] | Check change control records. |
| A.8.9 Cryptographic controls | Encryption applied where required. | [ ] | Validate encryption settings and key management. |
| A.8.10 Malware protection | Anti-malware controls implemented. | [ ] | Check AV tool dashboards. |
| A.8.11 Secure development | Secure coding, testing, and code review. | [ ] | Review development procedures. |

---

Use this checklist to demonstrate compliance during:

- Internal audits  
- Certification audits  
- Management reviews

# Information Security Policy

---

## 1. Purpose
The purpose of this Information Security Policy is to protect the confidentiality, integrity, and availability of information assets.  
This policy provides direction for managing security risks and ensures compliance with regulatory, legal, and contractual requirements.

---

## 2. Scope
This policy applies to:

- All employees, contractors, temporary staff, and third parties
- All data, information systems, applications, devices, and cloud services
- All locations where company information is processed or stored

---

## 3. Information Security Principles

| Principle | Definition |
|-----------|------------|
| **Confidentiality** | Information is accessible only to authorized individuals. |
| **Integrity** | Information is accurate, complete, and protected from unauthorized modification. |
| **Availability** | Information and systems are accessible when needed. |

---

## 4. Roles & Responsibilities

| Role | Responsibility |
|------|----------------|
| **Executive Management** | Owns the policy, approves changes, ensures resources are provided |
| **Information Security Officer / CISO** | Implements, monitors, enforces this policy |
| **IT Personnel** | Technical enforcement (access control, backups, security configurations) |
| **All Staff** | Must follow the security policy and report incidents |

---

## 5. Policy Requirements

### A. Access Control
- Access is granted based on **least privilege** and **need-to-know**.
- Multi-factor authentication (MFA) must be used where supported.
- User access is revoked upon termination or role change.

### B. Data Classification & Handling
All information must be classified into one of the following categories:

| Classification | Description |
|----------------|-------------|
| Public | Can be shared without restriction |
| Internal | For internal company use only |
| Confidential | Restricted to authorized personnel |
| Highly Confidential | Requires explicit approval and encryption |

Requirements:
- Confidential data must be stored in approved systems only.
- Encryption must be used for storage and transmission where required.

### C. Device & System Security
- All devices must have up-to-date security patches and antivirus software.
- Only authorized devices may connect to company systems.
- Personal devices must follow BYOD policies (if applicable).

### D. Incident Reporting
- Any **suspected or confirmed** security incident must be reported immediately.
- Use the Incident Management procedure for response and documentation.

### E. Password Standard (Minimum)
- Minimum length: _12 characters_
- Must include combination of characters (upper, lower, number, symbol)
- Password reuse is prohibited

### F. Third-Party / Vendor Security
- Vendors accessing company data must meet equivalent security controls.
- Security risks must be evaluated before granting access.

---

## 6. Compliance & Monitoring
- Compliance is monitored through internal audits, monitoring tools, and access reviews.
- Failure to comply may result in disciplinary action.

---

## 7. Review and Updates
This policy must be reviewed **annually**, or when significant organizational or regulatory changes occur.
